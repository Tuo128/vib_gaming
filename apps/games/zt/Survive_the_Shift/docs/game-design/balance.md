# 数值与难度

## 1. 平衡目标

- 普通敌人主要贡献数量感，不应普遍成为血牛。
- 玩家构筑每次升级都应产生可感知变化。
- 高难度通过组合、速度和机制增加压力。
- 正常难度允许首次接触割草游戏的玩家在几局内通关。

## 2. 初始基线

| 数值 | 初始建议 |
| --- | ---: |
| 玩家心态 | 100 |
| 玩家移动速度 | 280 px/s |
| 受伤无敌时间 | 800 ms |
| 基础武器伤害 | 1 |
| 基础攻击间隔 | 450 ms |
| 首次升级需求 | 8 XP |
| 标准单局时长 | 15 min |
| 撤离开门时间 | 1.5 s |

这些是原型基线，不是最终数值。

## 3. 难度配置

| 难度 | 敌人生成 | 敌人速度 | 敌人伤害 | 待办增长 |
| --- | ---: | ---: | ---: | ---: |
| 轻松摸鱼 | 0.75× | 0.90× | 0.75× | 0.70× |
| 正常上班 | 1.00× | 1.00× | 1.00× | 1.00× |
| 高强度加班 | 1.25× | 1.10× | 1.20× | 1.25× |

## 4. 加班指数

完成高强度加班后开放 1–5 级指数。每级优先加入一个规则变化：

1. 红点通知会短距离冲刺。
2. 精英敌人生成频率提高。
3. 摸鱼区更容易触发巡视。
4. Boss 增加一个技能阶段。
5. 撤离时电梯位置更远，最后一个需求获得强化。

## 5. 动态调节边界

可以根据性能动态降低敌人生成，但不应暗中根据玩家水平修改伤害或掉落。若提供辅助模式，应由玩家明确选择。

## 6. 数据记录

每局至少记录：

- 难度与角色
- 存活时间和结局
- 失败原因
- 最终等级
- 武器和被动构筑
- 总击杀
- 受到伤害
- 最高待办
- 摸鱼区停留时间

数据用于发现数值问题，不用于制造强制留存压力。
