# 技术框架

## 技术选型

- 构建工具：Vite
- 开发语言：TypeScript
- 游戏引擎：Phaser 3
- 物理系统：Phaser Arcade Physics
- 测试：Vitest
- 包管理：npm

## 选择 Phaser 的原因

- 适合桌面浏览器 2D 游戏。
- 内置场景、输入、音频、资源加载、摄像机和轻量物理。
- 便于实现大量简单敌人、对象池和碰撞检测。
- 可以逐步加入 WebGL 特效，同时保留 Canvas 降级能力。
- TypeScript 支持成熟，适合拆分数据、系统和表现层。

## 目录职责

```text
Survive_the_Shift/
├── prd.md
├── docs/
│   ├── technical/       # 技术决策、架构、开发约定
│   └── game-design/     # 玩法、系统、数值与内容规则
├── public/
│   └── assets/          # 最终游戏资源
├── src/
│   ├── main.ts          # Web 入口
│   ├── styles.css       # 页面容器样式
│   └── game/
│       ├── config/      # 游戏与难度配置
│       ├── core/        # 事件、状态和通用基础设施
│       ├── entities/    # 玩家、敌人、投射物
│       ├── scenes/      # Phaser 场景
│       └── systems/     # 可独立演进的玩法系统
└── tests/               # 单元测试
```

## 当前框架能力

- 启动、游戏和 UI 三个场景
- 玩家移动
- 敌人持续生成和追踪
- 鼠标瞄准、主动攻击和武器切换
- 玩家受伤、短暂无敌和游戏失败
- 击杀、经验、等级与简化成长
- 工作日倒计时
- 18:00 电梯撤离与胜利
- 难度配置入口
- 开始菜单与三档难度
- 不打断战斗的可视奖励三选一
- 独立定时生成、渐隐闪烁的血包
- 键盘、咖啡和文件粉碎机三种武器
- 红点通知、紧急需求、Excel 和“最后一个需求”
- 心态、待办、怒气、滑椅冲刺和茶水间
- 图形化状态条、固定武器槽、禁用态和悬浮说明
- 暂停、失败、重开和返回菜单

程序生成图形是开发期调试资源，最终会被正式美术资产替换。
